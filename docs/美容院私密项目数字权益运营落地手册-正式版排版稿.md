## 第一章 项目概述

### 1.1 项目背景

美容院私密项目具有高信任门槛、高客单、高售后敏感度的经营特征。传统门店经营模式中，普遍存在体验单转化不稳定、售后争议证据链不完整、员工提成口径不统一、老客复购与转介绍缺乏标准机制等问题。

为提升门店经营效率、客户生命周期价值与经营透明度，本项目拟基于 Nexus 搭建“会员、履约、分佣、风控”一体化数字权益运营体系。

### 1.2 项目定位

本项目定位为“消费权益数字化 + 门店经营协同数字化”。

- 顾客侧：会员等级、积分、项目券、预约优先权、服务记录等消费权益
- 员工侧：真实订单提成、复购激励、退款冲回
- 合伙人侧：白名单经营资格、治理协同、培训资格、广告额度等经营协同权益

### 1.3 项目目标

- 30 天内跑通：建档、下单、服务、确认、积分发放、会员升级、退款处理
- 60 天内跑通：老客复购、转介绍奖励、员工提成、售后冲回
- 90 天内跑通：合伙人白名单、KYC 审核、月度披露、第二家店复制准备

经营目标：

- 体验转疗程率提升
- 30 天复购率提升
- 退款率下降
- 投诉率下降
- 净回款提升

### 1.4 项目边界

1. 不面向不特定公众公开发售。
2. 不开放顾客侧公开交易。
3. 不承诺收益、升值、回购、分红。
4. 仅适用于合法生活美容/护理项目。
5. 触及医疗美容边界的项目不得纳入本系统直接经营。

## 第二章 组织架构与职责

### 2.1 项目组织结构

项目负责人统一统筹门店、技术、财务、合规四条线，店长负责门店执行，财务负责结算，合规负责人负责边界控制。

### 2.2 项目负责人职责

1. 确定项目边界、试点门店、核心指标。
2. 推进资源协调与阶段复盘。
3. 对经营结果和风险结果负责。

### 2.3 店长职责

1. 负责门店日常执行与人员管理。
2. 监督建档、成交、履约、回访、售后流程。
3. 组织周复盘和月复盘。
4. 上报异常退款、投诉及高风险员工情况。

### 2.4 咨询师职责

1. 负责顾客建档与需求识别。
2. 负责体验单成交与疗程包转化。
3. 负责预约安排与回访衔接。
4. 禁止夸大宣传和越界销售。

### 2.5 护理师职责

1. 负责服务履约。
2. 负责服务前、中、后记录。
3. 负责异常情况上报。
4. 负责顾客满意度反馈收集。

### 2.6 客服/私域运营职责

1. 负责 D1、D3、D7、D30 回访。
2. 负责社群激活、复购召回。
3. 负责老客转介绍承接。

### 2.7 财务职责

1. 负责净回款统计。
2. 负责退款审核和提成冲回。
3. 负责月度结算与对账。

### 2.8 合规负责人职责

1. 负责宣传话术审核。
2. 负责 KYC、披露、异常审查。
3. 负责隐私资料管理与风险上报。

## 第三章 业务模式设计

### 3.1 顾客生命周期设计

顾客生命周期分为：线索 -> 到店 -> 建档 -> 体验单 -> 疗程包 -> 年度管理 -> 老客复购 -> 转介绍。

### 3.2 SKU 结构设计

首期 SKU 建议分为四类：

1. 体验单：用于获客和信任建立
2. 疗程包：用于体验转化和周期管理
3. 年度管理包：用于高价值客户长期留存
4. 补偿包：用于售后安抚，不计业绩、不计提成

建议产品线：

- 私密修护线
- 紧致提升线
- 产后恢复线

### 3.3 会员与积分体系

顾客完成首单后自动纳入会员体系。门店按消费金额发放积分，积分仅用于抵扣或兑换，不可提现。会员按累计消费和订单次数升级，享受相应折扣与服务优先权。

### 3.4 员工提成体系

员工提成以真实完单为基础，退款和投诉成立后按规则冲回。提成体系包括咨询提成、服务提成、店长管理提成、转介绍提成。

### 3.5 合伙人权益体系

合伙人仅限白名单对象，需完成身份审核、合同签署和权限配置。合伙人权益仅用于经营协同，不向公众开放。

## 第四章 门店运营 SOP

### 4.1 新客建档 SOP

1. 登记顾客来源渠道。
2. 登记基础诉求与历史情况。
3. 标记是否存在风险提示。
4. 建立顾客档案编号。

### 4.2 风险筛查 SOP

1. 排查特殊生理阶段、明显炎症、术后恢复等情形。
2. 排查是否存在生活美容门店不可承接的情形。
3. 必要时建议就医或转诊。
4. 风险顾客不得强行成交。

### 4.3 体验单成交 SOP

1. 一次仅推荐一个体验项目。
2. 明确说明服务内容、预期、边界。
3. 不得使用治疗承诺和投资性表述。
4. 成交后立即预约服务时间。

### 4.4 服务履约 SOP

1. 服务前：签署知情同意，记录服务前评估。
2. 服务中：记录服务过程和异常情况。
3. 服务后：记录完成状态，顾客确认。
4. 完成后：系统发放积分，更新会员状态。

### 4.5 回访复购 SOP

1. D1 回访：感受与异常确认。
2. D3 回访：状态变化与满意度。
3. D7 回访：效果反馈与复购建议。
4. D30 回访：召回和老客转介绍激活。

### 4.6 售后退款 SOP

1. 先安抚，再核实。
2. 调取知情同意、服务记录、沟通记录。
3. 按规则判断退款、补偿或继续服务。
4. 进入退款流程后同步提成冲回。

### 4.7 投诉处理 SOP

1. 记录投诉来源和诉求。
2. 明确责任人和处理时限。
3. 留存完整处理记录。
4. 重大投诉升级至店长和合规负责人。

## 第五章 系统落地配置

### 5.1 门店配置

建立品牌实体、试点门店、管理员权限、门店资料、积分开关。

### 5.2 商品配置

全部项目按服务类商品建立，分层设置为体验、疗程、年度管理。

### 5.3 订单配置

统一采用服务类履约流程，保证每一笔订单都有开始、完成、确认、售后记录。

### 5.4 会员配置

按普通、VIP、VVIP、黑金四级配置等级与折扣。

### 5.5 积分配置

积分默认不可转让，设定返还比例、抵扣上限、有效期、补偿规则。

### 5.6 提成配置

设置咨询提成、护理提成、店长提成、转介绍提成及退款冲回规则。

### 5.7 KYC 配置

顾客默认不强制 KYC；合伙人、招商主管、核心店长按要求进行 KYC。

### 5.8 披露配置

按月输出经营简报，包含营收区间、退款率、投诉数、会员增长、广告投入。

## 第六章 风控与合规

### 6.1 宣传边界

不得使用“升值、回购、分红、保本、稳赚”等表述。

### 6.2 销售边界

不得将消费权益解释为投资产品，不得向公众进行经营权益推广。

### 6.3 服务边界

不得承接非法医美或超出门店资质范围的服务。

### 6.4 数据与隐私保护

所有敏感资料仅用于存证和内部管理，不得外泄。

### 6.5 合伙人准入边界

合伙人必须白名单、实名审核、签署协议、明确退出机制。

### 6.6 异常订单与预警机制

对高退款、高投诉、高争议订单和员工设置预警名单，定期复盘。

## 第七章 数据报表与经营复盘

### 7.1 日报

统计到店、建档、体验成交、服务完成、异常单数。

### 7.2 周报

重点关注体验转疗程率、复购率、退款率、投诉率、净回款。

### 7.3 月报

重点分析渠道结构、SKU 表现、会员增长、员工绩效、经营披露完成情况。

### 7.4 漏斗分析

按“线索-到店-建档-体验-疗程-复购”进行转化分析。

### 7.5 人员绩效分析

按咨询师、护理师、店长、客服分别评价。

### 7.6 经营决策机制

每周做小调整，每月做大调整，优先优化 SKU 结构和转化流程。

---

## 第八章 试点计划与验收标准

### 8.1 30 天目标

完成系统基础搭建，跑通门店完整订单闭环。

### 8.2 60 天目标

跑通积分、会员、提成、退款冲回和转介绍。

### 8.3 90 天目标

完成合伙人白名单、KYC、披露，并具备复制到第二家店的条件。

### 8.4 第二家店复制条件

1. 首店经营指标稳定。
2. SOP 完整。
3. 员工培训成型。
4. 风险事件可控。
